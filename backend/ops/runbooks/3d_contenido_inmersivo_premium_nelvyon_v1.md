# NELVYON Premium OS Standards — 3D y Contenido Inmersivo v1

**Parent:** [`nelvyon_master_operations_v1.md`](./nelvyon_master_operations_v1.md)

## NAME

3D y Contenido Inmersivo Premium OS Standards for NELVYON v1 (client delivery checklist — **paperwork layer only**)

## Purpose

This **3D Inmersivo Premium** template is **OS-facing delivery paperwork**: statuses, evidence, **format badges** (model_3d, animation_3d, AR, VR, product visualizer, interactive scene, motion_3d), and links — it does **not** connect game engines, cloud render farms, WebXR hosts, or 3D SaaS APIs.

Suggested OS / workspace touchpoints for honest verification:

- `/os/video-multimedia-premium/preview` — when motion/render narratives overlap traditional video delivery  
- `/os/contenido-copywriting-premium/preview` — when UX copy or narrative strings ship with immersive experiences  
- `/os/web-premium/preview` — when embeddable viewers or web shells are part of the story  
- `/help` — IP, accessibility (motion sickness), or platform policy escalations  

---

## Scope (minimum real)

- **Delivery discipline:** Format matrix and target platforms (web, app, headset class) documented before client sign-off.
- **Asset hygiene:** Poly budget, texture resolution, LOD, and export naming/versioning explicit — no silent “unlimited revisions”.
- **Performance honesty:** Frame budgets and device class targets described without claiming automated profiling from this surface.

---

## Limits (explicit non-goals)

- No Unity/Unreal/Blender render APIs, no Sketchfab/Threekit hooks, no AR cloud anchors from this template.
- Does not modify closed product fronts; does not open `/crm/deals`.
- No auto-generated CAD conversion or medical simulation claims from this OS surface.

---

## Premium checklist (3D & immersive delivery)

| Area | Premium expectation |
| --- | --- |
| READY | Golden path / `pnpm gate` discipline when 3D/immersive-related web surfaces ship in the same release train. |
| BRIEF & CONCEPT | Creative brief, reference boards, and interaction map documented — aligned with copy where applicable. |
| 3D MODELING | Topology, scale, and rig requirements honest — no phantom photogrammetry pipeline if not sold. |
| TEXTURING | PBR vs stylized path, UDIM/tile strategy, and brand palette traceability. |
| ANIMATION | Rig constraints, clip list, and loop/export rules explicit. |
| OPTIMIZATION | Draw calls, texture memory, and compression targets listed for agreed device class. |
| DELIVERY | glTF/USDZ/binary naming, checksums, and viewer embed posture — storage outside NELVYON unless product says otherwise. |
| REPORTING | Engagement/heatmaps claims limited to agreed definitions — template records expectations only. |
